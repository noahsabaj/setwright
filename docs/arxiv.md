# arXiv preflight and export

Setwright's arXiv Check is a reproducibility aid, not an arXiv submission or a
guarantee of acceptance. The PDF generated by arXiv from the uploaded archive is
always authoritative and must be reviewed there.

## Frozen input

A check freezes the exact project revision, main file, engine, runtime profile,
and preflight policy version. Any edit invalidates the result. The dependency
closure combines conservative static inspection with recorder output from a
clean `.fls` build; disagreement is reported and the union is inspected.

The check blocks at least:

- missing, absolute, outside-root, hidden, or symlinked dependencies;
- duplicate/case-colliding paths and archive-unsafe names;
- incompatible figure/font formats and shell-escape dependencies;
- PDF JavaScript or actions;
- missing or incompatible bibliography, index, glossary, or nomenclature
  outputs;
- a compile that needs files not present in the candidate archive.

Setwright-created code blocks use `listings`. An imported `minted` project is
eligible only when a compatible cache already exists for the selected runtime
and the clean build succeeds with shell escape disabled.

## Archive contents

The ZIP is deterministic: project-relative `/` paths are sorted, timestamps and
permissions are normalized, and only the dependency closure is included.
Eligible content includes source, assets, local classes/styles, bibliography
outputs (`.bbl`), indexes/glossaries (`.ind`, `.gls`, `.nls`), required fonts,
and compatible generated caches.

The export excludes `paper-settings.json`, `*.setwright-review`, VCS metadata,
editor/app state, backups, logs, SyncTeX, ordinary auxiliaries, and generated
PDFs. The preflight report is written beside the ZIP, never inside it. The MVP
does not emit `00README.json`.

## Clean-room proof

Setwright re-extracts the exact ZIP into a new sandbox and compiles from the
archive root with the pinned arXiv-compatible profile. The user sees that PDF,
the included/excluded manifest, diagnostics, source/runtime hashes, and any
remaining warnings.

The report conforms to
[`arxiv-preflight-report.schema.json`](../schemas/arxiv-preflight-report.schema.json).
It can say:

> Ready for arXiv upload — tested against the arXiv TeX Live profile

only when the exact archive clean-builds, no blocking finding remains, and the
user explicitly approves the displayed PDF. Readiness is scoped to the hashes
and policy recorded in that report.

## Upstream oracle and external gate

Linux release CI must run a commit-pinned checkout of the official
[arXiv submission-tools](https://github.com/arXiv/submission-tools) against the
same archive. The pin cannot be a moving branch or tag and must be reviewed when
updated. Comparison uses page count, extracted text, and rendered pages rather
than raw PDF bytes, which may contain nondeterministic metadata.

This scaffold does not yet record a reviewed submission-tools commit or a
successful real arXiv rehearsal. Both remain release gates, as does a final
review of arXiv's current
[submission guidance](https://info.arxiv.org/help/submit_tex.html).
